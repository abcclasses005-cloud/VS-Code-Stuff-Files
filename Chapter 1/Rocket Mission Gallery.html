Semantic HTML - Using tags that describe the meaning of content, not just its appearance What It Is Semantic HTML means using HTML tags that carry a clear meaning about the content they hold. Tags like header, article, and footer tell the browser and anyone reading the code exactly what kind of content is inside. Non-semantic tags like div and span say nothing about their content at all. Why It Is When code is meaningful, it becomes easier for everyone to understand. Other developers, search engines, and screen readers used by people with disabilities can all work with it. A page built with semantic HTML is easier to maintain, easier to find on Google, and accessible to a much wider audience. How It Works Students replace generic div tags with the correct semantic element for each section of the page. A header tag wraps the top of the page. A footer tag wraps the bottom. An article tag holds a self-contained piece of writing. The browser renders them all the same as a div visually, but the meaning they carry is completely different.

Div Tag - The generic container with no built-in meaning What It Is The div tag is a generic container element in HTML. It groups other elements together but carries no meaning of its own. Div is short for division. It is a plain box that holds content and is almost always used alongside CSS or JavaScript to create layout and interactivity. Why It Is Before semantic HTML existed, developers used divs for everything, including headers, footers, sidebars, and articles. Divs are still useful today when no semantic tag fits the purpose, or when a container is needed purely for layout. Understanding the div is essential because it shows students exactly what semantic tags improve upon. How It Works Students wrap content inside opening and closing div tags. The div creates a block-level container that stretches the full width of its parent. A class or id attribute is added to the div so CSS can target it for styling. In the activity, divs are used as starting placeholders that students upgrade to semantic elements.

This distinction is central to writing clean, professional HTML. Every web developer uses divs daily, and knowing when to use one versus a semantic tag is a sign of real skill. 3. Header and Nav - Marking the top of the page and its navigation links What It Is The header element defines the top section of a webpage or of a specific content block. It typically contains the site name, logo, and navigation links. The nav element sits inside or near the header and holds a group of navigation links that help users move between sections or pages of a website. Why It Is Having a dedicated header and nav element tells browsers and screen readers exactly where the site identity and navigation are located. Screen reader software announces the nav element to users, allowing them to skip straight to the content they need. Without these tags, every user has to manually scan the entire page. How It Works Students wrap the top section of the page in header tags. Inside the header, they add a nav element containing anchor tags with href attributes pointing to sections using id references like href='#health'. Each link scrolls the browser down to the matching section when clicked, creating smooth in-page navigation.

Main and Section - Wrapping primary content and organising it into themed chunks What It Is The main element wraps the primary content of the page, which is the part that is unique to that page and not repeated across other pages. There should only be one main element per page. The section element groups related content together under a common theme, like a chapter in a book. Why It Is The main tag tells browsers and screen readers exactly where the important content begins. Assistive technology can skip the header and jump straight to main, saving users with disabilities enormous amounts of time. The section tag breaks long pages into clearly themed chunks, making content easier to read and understand. How It Works Students wrap all the core content of the page, including the article, aside, figure, and iframe, inside a single main element. Inside main, they wrap the health topic content in a section tag with an id of health. The section acts like a labelled chapter, grouping the article and figure that belong to the same topic.

Section is like the different acts of the show. Each act has its own theme and is clearly separated from the ones before and after. 🌍 REAL-WORLD EXAMPLE: Wikipedia uses a main element to wrap the article content, separated from the navigation sidebar. News websites like BBC wrap their lead stories in a main element so assistive technology and search engines can identify the most important content immediately. Students who use them correctly produce pages that rank better in search engines, work properly with assistive technology, and are readable by other developers at a glance. These two tags transform a flat page into a structured document. 5. Article and Aside - Separating self-contained content from supplementary information What It Is The article element holds a self-contained piece of content that could be lifted out of the page and still make complete sense on its own, like a news article, blog post, or health guide entry. The aside element holds content that is related to the main content but not essential to understanding it, like a sidebar or tips panel. Why It Is Article and aside help browsers and search engines understand the difference between the core message and the supporting information. Search engines give the content inside an article tag more weight. The aside tag signals supplementary content, so screen readers can present it separately from the main reading flow. How It Works Students wrap the main health content, including the heading and paragraphs about exercise, inside an article element. The quick tips list is placed inside an aside element. The two elements sit side by side in the layout, but their semantic roles are completely different. One is the main story, the other is the helpful extra.

Aside is like the small factbox in the corner, useful extra information that adds context without interrupting the main story. Both matter, but one is primary and the other is supplementary. 🌍 REAL-WORLD EXAMPLE: BBC News wraps every news story in an article element. Wikipedia uses aside for the information box on the right side of articles, the one showing facts like population, country, and capital city. Students who use them correctly write pages that communicate their structure to search engines, accessibility tools, and other developers. Footer Element - Closing the page with a properly labelled landmark What It Is The footer element defines the bottom section of a webpage or content block. It typically contains copyright information, contact details, links back to the top of the page, and terms or conditions. Like the header, it is a landmark element that assistive technology specifically recognises. Why It Is The footer tag tells the browser and screen readers that the main content has ended and that what follows is closing or supplementary information. Without it, a div at the bottom of the page looks identical to every other div. There is no way to know it is the footer without reading all the content inside. How It Works Students replace the closing div at the bottom of the page with footer tags. Inside the footer, a paragraph holds the copyright symbol written as the HTML entity code followed by the year and site name. A link inside the footer can point to a section or page to help users navigate from the bottom of the page.

The footer is one of the most consistent design elements on the entire web. 🕵️WHY THIS MATTERS: The footer element completes the semantic structure of a webpage. Students who use it correctly produce fully accessible pages where every landmark, including header, main, and footer, is properly labelled for assistive technology and search engines. A page with a proper footer is a finished, professional webpage. 7. Figure and Figcaption - Pairing images with accessible captions semantically What It Is The figure element wraps self-contained media content such as an image, diagram, or video that is referenced from the main content. The figcaption element sits inside the figure and provides a visible caption or description for the media. Together they link an image to its explanation in a semantic way. Why It Is Before the figure element existed, images were placed inside divs with no semantic connection to their captions. Screen readers could not tell that the caption text belonged to the image. The figure and figcaption pairing solves this. Screen readers announce the image and its caption together, making visual content accessible to everyone. How It Works Students wrap the img tag inside a figure element. Immediately after the img, they add a figcaption element with a short description of what the image shows. The browser does not add any default styling to figure. It is purely semantic. The caption text appears below the image in normal document flow. 🎯 ANALOGY: Figure and figcaption are like a photograph in a textbook with a label underneath it. The photo is the figure, the visual content. The label beneath it is the figcaption, the explanation that tells you what you are looking at. Together they make complete sense. Separately they are just an image and some floating text. 🌍 REAL-WORLD EXAMPLE: Wikipedia uses figure and figcaption for every image in its articles. The photo of a person or place is the figure, and the small grey text underneath describing it is the figcaption. News websites use them for every news photograph. Even Google Images reads figcaption text to understand what an image shows. 🕵️WHY THIS MATTERS: Figure and figcaption make images a proper part of a webpage's content rather than decorative extras. Students who use them write pages where images are fully accessible, properly described, and semantically connected to the content around them. This is a professional standard followed by every major website on the internet. 8.
